# Repository Files

### Repo

- [ ] The `.gitignore` file prevents at least the following.
  - [Windows](https://github.com/github/gitignore/blob/main/Global/Windows.gitignore)
  - [MacOS](https://github.com/github/gitignore/blob/main/Global/macOS.gitignore)
  - [Linux](https://github.com/github/gitignore/blob/main/Global/Linux.gitignore)
- [ ] Add [LICENSE](https://docs.github.com/repositories/managing-your-repositorys-settings-and-features/customizing-your-repository/licensing-a-repository) file.

### Readme

- [ ] The `README.md` file is updated to latest exercise changes.
- [ ] Owner in `Copy Exercise` button is correct.
- [ ] Prerequisites set. They should be minimal.
  - [ ] Footer contains copyright notice.
  - [ ] Footer contains link to the license.
  - [ ] Footer contains link to code of conduct.

### Steps

- [ ] All learnings steps like `1-step.md` are in `.github/steps/`
- [ ] All images use absolute urls. They are NOT stored directly in the repository.
- [ ] Any sample files have intuitive code comments.
- [ ] Tense is consistent.
- [ ] Tone is consistent.
- [ ] Tone is casual, polite, active, and inspiring.
- [ ] Spell checker doesn't report issues.
- [ ] Grammar is regular English (not code talk).
- [ ] Acronyms are spelled out or explained.

### Workflows

- [ ] All grading and transition workflows are in `.github/workflows`
- [ ] Feedback workflows provide useful feedback for common user mistakes.
- [ ] Final workflow posts review
- [ ] Final workflow adds celebration message to readme.

### Sample Code

- [ ] If using a Codespace, it uses the minimum features to do the exercise.
- [ ] All files have intuitive code comments.
- [ ] Most/all boilerplate and setup is automated.
