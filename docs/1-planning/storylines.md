### Storylines

Storylines are a connective mechanism to provide consistency across the exercises. The goal is to provide realism, grounded practicality, a bit of fun, and (maybe one day) plot good enough to convince the learner to continue to the next exercise. 🤞

> [!IMPORTANT]
> We are actively determining storylines. If you have ideas, please share! 📣

### Tips

- Think of each exercise as an episode in a TV series. There is both a mini plot within the episode and a larger plot across the season.
- Try to use the same storyline for a learning path.
- Consider why your favorite movies and TV series work well and including similar ideas.
- Consider the target audience for the storyline and related exercises.
  - Example: GitHub Foundations is more likely to have newer developers.
- Use GitHub's history/characters and your imagination to make it interesting.
- Add plot twists, easter eggs, and other fun elements. Learning can be fun too! 🦄 🎉
- Try to use the output of an existing exercise as the starting point of your exercise. Connected stories are better than stories simply in the same universe. It's ok if they aren't perfectly aligned. Similar is enough to provide continuity.

### Try to avoid

- Do create exercises without story, essentially making a duplicate of the GitHub Docs.
- Avoid creating one-off unrelated exercises.
- Don't include polarizing, sensitive, or highly debatable topics.
